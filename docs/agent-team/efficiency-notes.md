# Efficiency Notes

## What makes this faster

- Small stories reduce context and review size.
- Lanes prevent unnecessary ceremony.
- Artifacts prevent repeated discovery.
- Skills isolate workflows so Claude loads only what is needed.
- Review output is severity-ranked and actionable.
- QA maps directly to acceptance criteria.
- Browser, risk, performance, and release gates are triggered only when relevant.

## What quality still requires

- Definition of Ready before engineering.
- Definition of Done before completion.
- Tests or explicit explanation when tests cannot be run.
- QA report for accepted stories.
- Human approval for protected work and release decisions.

## Efficiency metrics to track

- Cycle time per accepted story
- Rework rate
- QA fail rate
- Review blocker rate
- Human intervention rate
- Cost or token usage per accepted story
- Defect escape rate
